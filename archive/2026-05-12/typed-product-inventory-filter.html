Typed Product Inventory Filter

You're building the catalog layer for an e-commerce platform. Raw product entries arrive as `unknown` from a third-party supplier feed; your engine must validate them, apply typed filter criteria, and return a strongly-typed filtered inventory report — with zero `any`.

Goals

  • Implement `validateProduct` to narrow `unknown` → `Product` using a discriminated union result type — no `as` allowed.
  • Implement `matchesFilter` to apply each optional `FilterCriteria` field only when present, returning true only when all present criteria pass.
  • Implement `buildInventoryReport` to orchestrate validation + filtering in a single pass and return a fully-populated `InventoryReport`.
  • Handle edge cases: non-array input, empty tags arrays, out-of-range ratings, and non-string tag elements.

Hints

  • Use `typeof x === 'object' && x !== null` to satisfy R1 before accessing any properties — TypeScript will narrow the type for you.
  • For the `Category` union check, define a constant array `const CATEGORIES = ['electronics', ...] as const` and use `Array.prototype.includes` with a type predicate — this avoids any need for `as`.
  • In `matchesFilter`, short-circuit early: check each criterion with a guard like `if (criteria.inStockOnly && product.stock === 0) return false` to keep the logic flat and readable.
