Typed Product Inventory Filter & Summarizer

You're building the inventory management module for an e-commerce dashboard. Raw product entries arrive as `unknown` from a warehouse sync API; your module must validate them into strongly-typed `Product` records, filter them by availability and category, and return a fully-typed `InventorySummary` — with zero `any`.

Goals

  • Implement `parseProduct` to validate an `unknown` value into a typed `Product`, returning a `ParseResult<Product>` without throwing.
  • Implement `parseProducts` to batch-parse an `unknown[]`, silently skipping invalid entries.
  • Implement `filterProducts` to AND-filter a `Product[]` by category, status, and/or max price using `FilterOptions`.
  • Implement `summarizeInventory` to aggregate a `Product[]` into a fully-typed `InventorySummary` with per-category breakdowns, then wire all steps together in `processInventory`.
challenge.ts 
// Key types & main function signatures

export type Category = "electronics" | "clothing" | "groceries" | "furniture" | "toys";
export type AvailabilityStatus = "in_stock" | "low_stock" | "out_of_stock";

export interface Product {
  id: string;
  name: string;
  category: Category;
  priceUsd: number;    // must be > 0
  stockCount: number;  // must be >= 0, integer
  status: AvailabilityStatus;
}

export type ParseResult<T> =
  | { ok: true;  value: T }
  | { ok: false; error: string };

export interface FilterOptions {
  categories?:  Category[];
  statuses?:    AvailabilityStatus[];
  maxPriceUsd?: number;
}

export interface InventorySummary {
  totalProducts:   number;
  totalStockValue: number;
  byCategory:      Partial<Record<Category, CategorySummary>>;
  lowOrOutOfStock: Product[];
}

// Pipeline entry point
export function processInventory(raw: unknown[], options: FilterOptions): InventorySummary
Hints (click to reveal)

Hints

  • Use a `Set<Category>` with `.has()` to narrow `unknown` strings into the `Category` union — TypeScript will accept this as a type guard when the set is typed correctly.
  • For `byCategory` in `summarizeInventory`, initialise each new category entry with all three `AvailabilityStatus` keys set to `0` so the `Record<AvailabilityStatus, number>` shape is always complete.
  • When checking `FilterOptions` criteria, treat an empty array (`length === 0`) the same as omitted — only apply the filter when the array is non-empty.
